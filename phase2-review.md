# Phase2 API Review（採用判断用サマリ）

## 方針
- 本書は「採用判断・監査・縮退設計」のためのサマリ。実装詳細は /docs/apis/ に置く
- API結果は判断の代替にしない（要点整理・差分提示のみ）
- “最新保証”をしない。必ず参照時点（as-of）と公式確認導線で閉じる
- Terms/License 未確認の候補は HOLD とし、台帳投入は行わない。

## 横断ルール（全API共通）
- as-of（取得日時）を必ず内部ログに残す／ユーザーには必要時のみ提示
- キャッシュする場合はTTLと規約適合を明記
- “公的データを元に整理すると”の表現に統一（断定禁止）
- APIが落ちてもUXが成立すること（最悪：公式リンク案内のみ）
