# Document Authority Model（実装なし）

## 1. 権威レイヤー定義
Layer 1:
- 対象文書: `PARENTY_SSOT.md`
- 役割: SSOT（憲法）
- 持つ権威: 定義の唯一の正
- 禁止される行為: 他文書での再定義・拡張・置き換え

Layer 2:
- 対象文書: `PolicyUxAdminMatrix.md`
- 役割: Matrix（横断契約）
- 持つ権威: ドメイン横断の接続点の唯一の正
- 禁止される行為: SSOTを越える定義・独自接続の追加

Layer 3:
- 対象文書: `UX_STATE_MAP_7.md` ほか下位仕様
- 役割: 下位仕様（使用仕様・境界の固定）
- 持つ権威: SSOT/Matrixの参照範囲での使用条件
- 禁止される行為: SSOT/Matrixの再定義・拡張・例外化

Layer 4:
- 対象文書: `Guides.md` / `Runbook.md` / `Observability.md` / `APIRegistry.md` ほか派生文書
- 役割: 派生文書（参照導線・説明）
- 持つ権威: 権威なし
- 禁止される行為: 定義・判断・列挙を正として扱うこと

## 2. 参照ルール
上位→下位は参照と制約付けのみ許可し、下位→上位は参照のみ許可し再定義は禁止する。同一レイヤー間は説明目的の参照のみとし、権威の移動や統合は行わない。派生文書間の相互参照は許可するが権威はゼロとし、正の根拠にはならない。

## 3. 禁止行為
- 派生文書で定義・判断・列挙を正として扱うこと
- 例・チェックリスト・手順をテンプレとして実装すること
- SSOT / Matrix を経由せずに仕様判断を行うこと
- 複数文書を統合して新しい解釈を作ること
- 文書の役割を超えた記述を追加すること

## 4. 本文書の位置づけ
本文書は仕様ではなく、仕様の内容を定義しない。本文書は仕様の読み方と力関係のみを定義する。本文書は SSOT 構造の変更が発生した場合にのみ更新対象となる。

## 5. 配置と参照
- ファイル名: `DOCUMENT_AUTHORITY_MODEL.md`
- SSOT_INDEX 参照位置: `SSOT_INDEX.md` の関連リンク
